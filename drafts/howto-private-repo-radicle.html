How to set up a shared private repository with radicle?

Posted on November 4, 2024

I fell in love with Radicle, so much so that I am trying to use for every project I start. The hardest part is convincing friends I am working with to use it: letâ€™s face it, while radicle is awesome it also has a somewhat steep learning curve. Even though the team has put effort on documenting and providing useful guides, those guides are mostly tutorials (see 4 quadrants theory of documentation) but when doing things one needs howtos.

This short document is such a practical howto, aiming at explaining the steps two people need to start collaborating one a piece of privately using Radicleâ€™s peer-to-peer infrastructure. The end goal is that those two people should each be able to use standard git commands to share whatever changes they are making to a code repository, without the need of a centralised server.

Set up

We assume the following setup:

  • Each user, say Alice and Bob, has a working installation of Radicle on their local machine
  • There is an accessible seeder machine one or both of the users have access to, named Charlie for the purpose of this Howto.
    • This step might not be necessary if both usersâ€™ machine can connect to each other directly, either through standard TCP (eg. they are on the same network) or using TOR

Creating the repository

Alice creates a git repository to share work with Bob:

mkdir secret-project
cd secret-project
git init
echo "Secret project with Bob" > README.md
git add README.md
git commit -m "Initial commit"

Then she initialises this repository as a radicle repository:

rad init --private

Sharing it with public seed and Bob

To allow Bob to commit directly to this repository, Alice needs to add him as a delegate.

rad id update --title "Allow delegate to Bob" --allow "did:key:<bob's id>" --delegate "did:key:<bob's id>"

And for both parties to be able to collaborate they need the public seeder to be able to seed the repository, so that Bob can clone it and the seeder can serve as a relay between Alice and Bob:

rad id update --title "Allow public seeder" --allow "did:key:<seeder id>"
